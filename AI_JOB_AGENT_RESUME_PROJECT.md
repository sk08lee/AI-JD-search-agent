AI Agent 实习岗位搜索助手（产品原型）

## 简历正式版

**AI Agent 实习岗位搜索助手**  
项目方向：AI Agent · RAG · MCP · Playwright · LLM 应用产品设计

- 基于 LLM + MCP + RAG 学习底座，**独立完成场景选型与产品化**，聚焦计算机/AI 方向研究生 **实习岗位 JD 研究** 垂直场景，输出 PRD、Demo Case 与合规边界文档。
- 接入 **11 家大厂公开实习入口**（字节、腾讯、阿里、美团等），设计 Playwright 动态页抓取 + **Agent Fetch 降级** 方案，保障抓取失败时仍可生成报告。
- 定义 **两章报告结构**（官网 JD 事实 + 本地知识库归纳），规则包括：岗位标题含「实习」过滤、每公司 1 条、0 匹配跳过、用户输入关键词驱动搜索。
- 完成 **Web UI + 阿里云 ECS Docker 部署**，报告由 API 直接返回、**一键复制**，无需写本地文件。
- 明确合规边界：仅读公开招聘官网，不绕过 BOSS/拉勾等平台登录或反爬。

## 面试介绍话术（3 分钟）

这个项目源于我学习 LLM、RAG 和 MCP 的过程。学完后我没有停在 Demo，而是找了一个自己真实会用的场景：**准备 AI 产品经理实习投递**。

我发现实习 JD 分散在各家大厂官网，搜索引擎只能给链接，没法直接告诉我该补哪些能力。所以我定义了一个 Agent：**用户输入岗位名，系统从 11 家公开实习入口抓取 JD，再结合本地岗位知识库，生成两章报告**——第一章是官网事实，第二章是能力归纳和简历建议。

产品上有几个关键取舍：第一，V1 **只做实习**，过滤非实习岗位；第二，只做 **公开合规数据源**；第三，Playwright 抓取不稳定时设计了 **Agent + Fetch 降级**；第四，报告 **直接在 Web 页面复制**，不写文件。

最后我把它部署到了阿里云，做了 Web 界面，可以完整演示从输入到复制报告的全流程。这个项目体现的是我对 Agent 产品化的理解：不只是调模型，而是任务拆解、数据源策略、降级设计和交付体验。
