# AI 产品经理简历项目经历

## 推荐项目名称

AI Agent 岗位搜索助手产品原型

## 简历正式版

**AI Agent 岗位搜索助手产品原型**  
项目方向：AI Agent、RAG、MCP 工具调用、岗位搜索、LLM 应用产品设计

- 基于 LLM + MCP + RAG 技术底座，设计面向应届生和实习求职者的 AI Agent 岗位搜索助手，帮助用户检索公开岗位信息并生成岗位需求报告。
- 将求职信息搜索任务拆解为岗位关键词解析、公开来源读取、本地岗位知识库检索、岗位类型分类、要求抽取和 Markdown 报告生成等环节。
- 设计实习、校招、社招三类岗位分类体系，围绕岗位职责、任职要求、技能关键词、项目经历和简历优化建议输出结构化分析。
- 引入 MCP Fetch 和 Filesystem 工具调用能力，验证 Agent 从信息检索到结果交付的端到端产品闭环。
- 明确招聘平台合规边界，V1 优先使用公司招聘官网和公开网页，招聘软件信息仅支持公开链接或手动导入。

## 简历精简版

**AI Agent 岗位搜索助手产品原型**  
项目方向：AI Agent、RAG、MCP、AI 产品设计

- 设计面向应届生和实习求职者的 AI Agent 岗位搜索助手，支持按实习、校招、社招分类生成岗位需求报告。
- 基于 RAG 复用本地岗位知识库，结合 MCP Fetch 读取公开招聘页面，汇总岗位职责、技能要求和项目经历要求。
- 输出 PRD、Demo Case、产品指标和合规边界说明，展示从技术能力到求职场景产品化的设计过程。

## 面试介绍话术

这个项目是我基于 LLM + MCP + RAG 技术底座设计的 AI Agent 岗位搜索助手。它的目标不是做一个普通搜索工具，而是帮助求职者把分散的岗位信息整理成可行动的岗位需求报告。

我把用户任务拆成几个环节：先解析用户想了解的岗位方向，比如 AI 产品经理；然后检索本地岗位知识库，并通过 MCP Fetch 读取公开招聘页面；接着按实习、校招、社招三类归纳岗位要求；最后生成 Markdown 报告，给出高频能力、技能关键词、项目经历要求和简历优化建议。

这个项目能体现我对 AI Agent 产品的理解：RAG 解决“已有知识如何复用”，MCP 解决“如何连接外部信息源和工具”，Agent 解决“如何把搜索、分析和报告交付串成任务闭环”。

## 项目亮点

- 场景明确：聚焦 AI 岗位搜索和求职准备。
- 用户明确：面向计算机专业研究生、应届生和实习求职者。
- 分类明确：实习、校招、社招三类岗位需求对比。
- 边界明确：只读取公开网页，不绕过招聘平台登录或反爬。
- 交付明确：输出可直接用于简历优化和面试准备的岗位需求报告。
