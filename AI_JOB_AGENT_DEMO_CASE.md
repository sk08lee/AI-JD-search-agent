# Demo Case：AI 产品经理岗位需求报告生成

## Demo 目标

验证 AI Agent 能否围绕求职场景完成一个端到端任务：

**根据用户输入的岗位方向，结合本地岗位知识库和公开招聘网页，生成一份岗位需求分析报告，并保存为 Markdown 文件。**

## 用户输入

```text
我是一名计算机专业研究生，想投 AI 产品经理实习岗位。
请帮我搜索公开招聘信息，总结 AI 产品经理实习岗位常见要求，
并按技能、项目经历、工具能力、业务理解、加分项输出一份 Markdown 报告。
```

## 系统流程

1. Agent 解析岗位方向：AI 产品经理。
2. Agent 识别主要岗位类型：实习，同时保留校招和社招对比。
3. RAG 检索 `knowledge/jobs` 中的岗位需求知识库。
4. Fetch MCP 尝试读取公开招聘官网或公开岗位页面。
5. LLM 汇总岗位职责、任职要求和技能关键词。
6. Agent 调用 Filesystem MCP 保存报告到 `output/ai-product-manager-job-demand-report.md`。

## 输出报告示例结构

```md
# AI 产品经理岗位需求报告

## 1. 岗位搜索概览

## 2. 实习 / 校招 / 社招岗位差异

## 3. 高频能力要求

## 4. 高频技术关键词

## 5. 常见项目经历要求

## 6. 对计算机研究生的简历优化建议

## 7. 本项目可包装的匹配点

## 8. 信息来源和待补充来源
```

## 产品价值

- 帮助求职者快速理解岗位真实要求。
- 帮助应届生区分实习、校招和社招的能力门槛。
- 帮助用户把自己的 AI 项目经历映射到岗位要求。
- 将原项目的 RAG、MCP、工具调用能力落到清晰的求职场景。

## 验收标准

- 报告必须按实习、校招、社招三类组织信息。
- 报告必须包含岗位能力要求和简历优化建议。
- 如果公开网页读取失败，必须明确标注来源不可访问。
- 不允许编造具体公司岗位正在招聘的信息。
- 输出文件必须保存到 `output/ai-product-manager-job-demand-report.md`。
