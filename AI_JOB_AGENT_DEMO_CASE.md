# Demo Case：AI 产品经理实习岗位需求报告

## Demo 目标

验证 AI Agent 能否围绕 **实习求职** 场景完成端到端任务：

**用户输入岗位名称后，系统从 11 家大厂公开实习入口检索匹配 JD，结合本地岗位知识库，在 Web 页面生成可直接复制的两章 Markdown 报告。**

## 演示环境

- Web UI：`http://localhost:9000` 或已部署的 ECS 地址
- 关键环境变量：`ENABLE_AUTO_CAREER_FETCH=1`、`CAREER_INTERNSHIP_ONLY=1`、`CAREER_FETCH_MAX_SOURCES=11`

## 用户输入（Web 表单）

| 字段 | 示例值 |
|------|--------|
| 岗位方向（可选） | 产品 |
| 具体岗位名称 | AI产品经理 |

等价自然语言任务：

```text
我是一名计算机专业研究生，想投 AI 产品经理实习岗位。
请从公开实习招聘入口检索匹配岗位，结合本地知识库，
输出可直接复制的 Markdown 实习岗位需求报告。
```

## 系统流程

1. 加载 `knowledge/sources/career_portals.json` 中 **11 家大厂实习入口**。
2. 以用户输入的 **「AI产品经理」** 为关键词，在各官网搜索页检索岗位。
3. **Playwright** 渲染动态招聘页，提取 JD（公司、岗位名、地点、职责、条件、URL）。
4. 过滤非实习岗位（标题须含「实习」）；每公司最多保留 1 条；0 匹配则跳过。
5. Playwright 无结果时，触发 **Agent + Fetch MCP** 兜底读取公开页面。
6. **RAG** 检索 `knowledge/jobs/ai_product_manager/` 本地岗位知识库。
7. **LLM** 合并官网 JD 与知识库，生成两章 Markdown 报告。
8. API 返回报告正文 → Web 页面展示 → 用户 **一键复制**（不写本地文件）。

## 11 家实习入口

字节跳动 · 腾讯 · 阿里巴巴 · 美团 · 百度 · 华为 · 京东 · 网易 · 小米 · 商汤科技 · 科大讯飞

## 输出报告结构

```md
# AI产品经理实习岗位需求报告

## 一、公开招聘官网岗位（核心 JD）

### 字节跳动
- **岗位名**：……（日常实习）
- **地点**：……
- **岗位职责**：……
- **招聘条件**：……
- **来源**：[岗位详情 URL]

（其他有匹配的公司依此类推；0 匹配的公司不出现）

## 二、结合本地岗位知识库归纳（能力要求与求职洞察）

### 共性能力要求
（综合第一章官网条件提炼）

### 技术栈与工具倾向
……

### 常见项目经历
……

### 简历优化建议
……
```

**不包含**：校招/社招对比、信息来源附录、保存路径说明。

## 产品价值

- 帮助实习求职者 **5 分钟** 了解某岗位在各公司的真实 JD 要求。
- 将分散在 11 家官网的实习信息 **结构化** 为可对比、可复制的报告。
- 第二章结合 RAG 知识库，给出 **可执行的简历优化方向**。
- 将 LLM + RAG + MCP + Playwright 能力落到清晰的垂直求职场景。

## 验收标准

- [ ] 仅输出 **实习岗位**；非实习岗位被过滤
- [ ] 搜索关键词来自 **用户输入**，非硬编码岗位名
- [ ] 报告包含固定 **两章**结构，顺序正确
- [ ] 第一章每条 JD 有 **来源 URL**；0 匹配公司不出现
- [ ] 第二章 **综合** 官网条件与知识库，非原样堆砌
- [ ] 报告可在 Web 页面 **直接复制**；不依赖本地 `output/` 文件
- [ ] 不编造 context 中未出现的公司在招岗位
- [ ] 公开页面不可访问时标注原因，不虚构 JD

## 演示检查清单

1. 打开 Web UI，输入「AI产品经理」，点击生成
2. 等待报告返回（含 Playwright 抓取，可能需 1–3 分钟）
3. 确认第一章有至少 1 家公司实习 JD（视抓取成功率）
4. 确认第二章有能力归纳与简历建议
5. 点击「复制报告」，粘贴到 Markdown 编辑器验证格式

## 已知局限（Demo 说明）

- 部分站点（腾讯、美团、京东等）Playwright 适配仍在优化，可能 0 匹配
- 字节、小米等站点相对稳定；失败时会走 Agent fallback
- 报告质量依赖 LLM 与 Embedding API 可用性
